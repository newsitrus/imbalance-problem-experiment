# 1. Introduction

Class imbalance is a state when dataset used in a classification problem is skewed towards one of the classes. Since standard classification methods aim to minimize the error rate, they weight the classes with more observations higher. Real life data-sets usually have skewed class distribution. Therefore, if the model is not properly developed, minority class observations end up being overlooked. Hence, class imbalance problem has become an increasingly hot topic in recent years.

Classification methods were developed in order to determine the order in the observation data and to categorize the sample based on this order (Lavine $\&$ Mirjankar, 2012). Classification is a kind of pattern recognition method and assigns each input value to a value in the vector of "classes". In almost all of the real data, the numbers of observations pertaining to classes show a skewed distribution, which, then is called class imbalance. However, the term class imbalance is used for situations where there is a significant imbalance rate. This is called between-class imbalance and is a common situation (He & García, 2009). Another type of class imbalance is called within-class imbalance. There are rare observations within a class of this type of imbalance. In this study, class imbalance term will be used to express between-class imbalance. In the case of class imbalance, the classifiers show bias towards the majority class. The minority class is typically referred to as "positive" and the majority class as "negative".

Resampling methods are pre-processing methods in which the dataset is modified so that the classes of observations in the data-set become more balanced (Fernández et al., 2018a). Past studies have shown that modeling by achieving class balance with resampling methods is a useful solution approach (Chawla et al., 2008; Estabrooks et al., 2004; García, Sánchez, & Mollineda, 2012).

Many methods have been developed in literature to tackle the class imbalance problem in recent years (Charte et al., 2019; Loyola-González et al., 2016; Wong et al., 2020; Yijing et al., 2016). These methods can generally be gathered under the following three groups:

Cost-sensitive classification Ensemble algorithms Resampling methods

A cost matrix which represents the cost of predicting the classes wrong is used in cost-sensitive classification. In this matrix, false positive and false negative states are taken into account. The costs of two classes are usually not the same in real applications (Zhang, 2019). Examples of these conditions include disease diagnoses. Diagnosing a sick person as "healthy" is more costly than diagnosing a healthy person as "sick". Therefore, balance is achieved by weighting classes using cost matrix. Ensemble algorithms are used to create a strong classifier using multiple weak classifiers. Researchers have used ensemble algorithms to handle class imbalance problem before (Galar et al., 2012; Oza & Tumer, 2008).

In resampling methods, minority class set is oversampled and/or majority class set is undersampled until balance is achieved and thus, bias caused by class imbalance is reduced (Das et al., 2015). Resampling methods can be categorized into three groups. These are undersampling, oversampling, and hybrid methods that use both oversampling and undersampling. In undersampling methods, observations are discarded from the observations of the negative class in order to achieve balance. The simplest form of undersampling is random undersampling (RUS) method. In this method, observations to be discarded from the negative class are randomly selected. Since valuable information is lost during undersampling, different sample selection approaches have been proposed in the literature. Zhang and Mani (2003) proposed a method based on the nearest neighborhood algorithm called NearMiss which has five different types. Other approaches based on nearest neighbors include Condensed Nearest Neighbor (CNN) (Hart, 1968), Edited Nearest Neighbor (ENN) (Wilson, 1972), and Tomek Link (Tomek, 1976) methods. These methods are based on nearest neighbors and detect noisy data from both positive and negative classes. However, they are also used to undersample negative data-sets in order to achieve balance as undersampling methods. They can be used to discard observations from both classes or only from negative classes. Apart from these, there are also many proposed undersampling methods which use optimization to determine the observations to be discarded (Ha & Lee, 2016; He & García, 2009; Triguero et al., 2015; Yu et al., 2013). The aim of the oversampling methods is to increase the number of positive class data-set observations and achieve a more balanced data-set. There are studies examining the successful results of oversampling methods compared to undersampling (García et al., 2020). However, they are limited, which they also mention, and one must be careful to reach such conclusion. Just like RUS, random oversampling (ROS) is another simple way to do this. ROS is the process of taking samples from positive class observations and then, adding them to the data-set (He & García, 2009). This method is simple to implement and understand. In ROS, repetition of the same observations causes the classifier rules to be very specific since classifier concentrates on certain areas. Therefore, excessive conformity of the classifiers with the train data-set can be observed. This indicates that the test data-set may demonstrate poor performance. Chawla et al. (2002) proposed an alternative method, which later became very popular, known as synthetic oversampling minority examples (SMOTE). SMOTE randomly generates synthetic minority class samples on the link between the minority class instance and its specific number of neighbors with the same class. Since new data samples are in a defined neighborhood, instead of data space, it focuses on the feature space (Fernández, García, Herrera, & Chawla, 2018). SMOTE has some disadvantages. The first disadvantage is that the newly produced synthetic data can be generated in the majority class region. This causes the generation of noise patterns that are not normally present. The second disadvantage concerns the links with the neighbors. The number of links has a fixed $k$ value for each sample. It is not possible for the number of neighbors to take different values from sample to sample. In addition, the qualities of the neighbors to which the link is established are not taken into account when generating synthetic data. In this study, we will examine the mentioned disadvantages of the SMOTE method and propose a new approach to solve it.

The primary emphasis in this article is on ROS and SMOTE. Section 1 gives description of the problem and literature review on class imbalance, noise detection and boosting. Section 2.1 shows SMOTE algorithm and illustrates its shortcomings and Section 2.2 examines related works. Section 3 introduces a new method that makes use of boosting and observation neighborhoods to solve the shortcomings of the SMOTE method. Section 4 provides information about experimental set-up, simulated and benchmark data-sets with imbalanced classes, resampling algorithms, classifiers and performance metrics. Section 5 gives experimental results and illustrations and discusses the results. Section 6 concludes the study and remarks its importance.

# 2.2. Related work

Various studies conducted to improve SMOTE. Chawla et al. (2003) proposed an approach consisting of a combination of the boosting procedure and the SMOTE algorithm called SMOTEBoost. In the SMOTE-Boost method, positive class synthetic observations are generated in each weak classifier and the weight updating for skewed distributions changes indirectly. Han et al. (2005) proposed two new over-sampling methods based on the SMOTE called BL-SMOTE1 and BL-SMOTE2. In these methods, data is created only in the border regions between classes. In experimental studies, it was observed that the methods proposed reached higher true positive rate and $F _ { 1 }$ value compared to SMOTE and ROS method. He et al. (2008) proposed ADASYN method, another SMOTE based synthetic sampling approach for imbalanced data-sets. The underlying concept is to use weights for minority class observations based on learning difficulties. In this method, more data is created for observations that are difficult to learn, and less data for observations that are easy to learn. Bunkhumpornpat et al. (2009) categorized instances based on their safe level ratio and generated synthetic instances in safer regions using a method called Safe-Level-SMOTE. Seiffert et al. (2010) proposed a hybrid method that combines RUS and the boosting algorithm to solve the class imbalance problem. The method follows the same procedure as the SMOTEBoost method. It applies RUS instead of just SMOTE. Maciejewski and Stefanowski (2011) introduced LN-SMOTE which uses the information on local neighborhoods. Gao et al. (2011) used SMOTE and radial basis function classifier together with particle swarm optimization (PsO) to handle class imbalance problem. First, they used SMOTE to balance the data-set and then PSO to determine kernels. García et al. (2012b) investigated neighborhoods of examples by three different definitions, nearest centroid, graph and relative neighborhood, and proposed three SMOTE variant method based on them. Bunkhumpornpat et al. (2012) examined density-based clusters in the datasets using DBSCAN and determined the shortest path from positive cases to cluster centroids. Then synthetic instances are generated on this path using their new method DBsMOTE. Verbiest et al. (2014) used data cleaning using fuzzy rough prototype selection before and after SMOTE to improve data quality. Their experiments show that their method performs well in the presence of noise. Sanguanmak and Hanskunatai (2016) proposed a hybrid resampling method named DBSM. This method first undersamples the dataset using DBSCAN then uses SMOTE to oversample. Bunkhumpornpat and Sinapiromsaran (2017) proposed a new method called DBMUTE. This method, similarly to DBsMOTE, determines density-based clusters using DBSCAN. They first undersampled majority classes in overlapping regions using majority under-sampling technique and then generated new instances on the shortest path to cluster centroids. Ma and Fan (2017) proposed a new resampling method which combines clustering using representatives and SMOTE for random forest algorithm. Results showed that hybrid algorithm performs better than original random forest. Douzas et al. (2018) used k-means clusters to determine safe areas and oversampled using SMOTE in safe areas to combat within-class imbalance. Douzas and Bacao (2019) investigated the geometric structure of the data-set and determined a region around selected instances using SMOTE. Cheng et al. (2019) explored the noise and fixed neighborhood parameter of SMOTE. First, they proposed a noise removal approach which compares instance density of different classes. Then assigned different number of neighborhoods based on global number of neighborhoods for resampling. There are also more modified versions of SMOTE in the literature (Nekooeimehr & Lai-Yuen, 2016; Sáez et al., 2015; Santos et al., 2015; Susan & Kumar, 2019).

# SMOTE with boosting (SMOTEWB)

SMOTE generates data from all positive observation points in the same way. In cases where positive class observations, which are taken into account, are far from other positive observations and are among negative observations, the generation of data between neighbors may produce noise. This creates issues not usually present in the data-set.

SMOTE has two main problems. The first is that, as mentioned before, it can generate noise. Another problem is that the number of neighbors and properties of them to be connected for synthetic data production is not taken into consideration. The proposed method aims to overcome "only" the problems caused by SMOTE. Therefore, it can be used inside other SMOTE-based resampling algorithms. Other issues related to classification (variable selection etc.) are not addressed.

To eliminate the first of these two problems, noises among positive and negative observations must be detected. In order to do this, we propose a new approach, which uses boosting procedure of Adaboost algorithm (Freund & Schapire, 1996) in Algorithm 2. The updated weights of the observations obtained by the adaptive boosting method show the predictability measure of the observations. Keeping these weights into consideration, one can infer which observation is well predictable and which observation is poorly predictable. Fig. 2 shows a sample two-dimensional data-set and weights obtained by applying adaptive boosting to this data-set. The weights of positive observations are shown to be greater than those of negative observations. This is because positive results are difficult to predict in the case of imbalanced classes, and the model would display bias towards the negative class. Average weight of positive observations is 0.00397 and average weight of negative observations is 0.00060. From this result, it is clear that positive observations are more difficult to predict, just as expected. With the help of these weights, noises in both positive and negative classes can be detected. In Fig. 2(c), observations are sized proportional to the weights obtained. Positive and negative observations reaching the area of other class observations are shown to be easily understood through adaptive boosting. Observations with weight over $1 / n$ , which is the initial weight, are more difficult to predict. It can be assumed that observations with a weight less than $1 / n$ are relatively easy observations. Since class imbalance is derived from the negative class, addressing the two class observations in this manner would not be right. By determining a new threshold value for weights, observations can be labeled as noise or not noise. In this case it is not possible to specify an absolute threshold. In this study threshold values in equation (2) and (3) are used.

$$
T ^ { p o s } = \frac { 1 } { n } \left( \frac { 2 \times n ^ { n e g } } { n } \right) = \frac { 2 \times n ^ { n e g } } { n ^ { 2 } }
$$

$$
T ^ { n e g } = \frac { 1 } { n } \left( \frac { 2 \times n ^ { p o s } } { n } \right) = \frac { 2 \times n ^ { p o s } } { n ^ { 2 } }
$$

here, $T ^ { p o s }$ is the threshold for positive class observations and $T ^ { n e g }$ is the threshold for negative class observations. Starting weight for adaptive boosting is $1 / n$ for each observation and the parts in parenthesis are the ratios of the observation excess of the opposite class. If classes are fully balanced, then $( 2 \times n ^ { n e g } ) / n = ( 2 \times n ^ { p o s } ) / n = 1$ and both threshold values become $1 / n .$ If class distribution is skewed and threshold is taken as $1 / n$ , more positive observations are actually treated as noise, since positive observations will have a higher weight. In order to eliminate bias caused by imbalance, it is reasonable to increase the positive observation threshold value depending on the imbalance rate and to decrease the negative observation threshold value in the same way. Since the number of negative observations is more than $n / 2$ when there is class imbalance, $2 \times n ^ { n e g } > n$ and $T ^ { p o s } > 1 / n$ Likewise, $2 \times n ^ { p o s } < n$ and $T ^ { n e g } < 1 / n$ .Using Eqs. (2) and (3), more realistic noise detection can be achieved in positive classes and negative classes. This change in threshold value is considered reasonable, since it only takes into account the imbalance rate and there is a problem related to the class imbalance. Noise detection procedure is given in Algorithm 3.

To eliminate the second problem of SMOTE, the number of links to be established for the observations must be calculated correctly. Unlike SMOTE, SMOTEWB do not require a $k$ parameter determined by the user. In the proposed method, first of all, the maximum number of nearest neighbors is determined based on the size of the train set as in Equation (5).

$$
k ^ { m a x } = \left\lfloor \frac { n ^ { n e g } } { n ^ { p o s } } \right\rfloor
$$

This ratio is the rounded down number of synthetic samples to be produced per average observation to ensure class balance. In standard SMOTE, when the Equation (5) is low (low imbalance) and the number of $k$ neighbors is high, less observation is randomly assigned to too many neighbors. In this case, excessive links with neighbors cause meaningless scattering of new observations. When the same equation (5) is high (high imbalance), it means that too much data will be derived from a positive observation, and the lack of neighbor links means that the new data will be concentrated in a small region which lead the model to overfit. It is intended to overcome this problem by determining the maximum number of neighbor bonds from Equation (5).

Once $k ^ { m a x }$ is determined, instead of assigning the same neighbor number for each positive observation, a different $k _ { i }$ neighbor number is determined for each positive observation using Algorithm 4. Positive observations previously labelled as "noise" and "not noise" will be divided into three groups as "good", "bad", and "lonely" after this algorithm.

When the distances to $k ^ { m a x }$ neighbor in the non-noise subset of positive observations is sorted, nearest first non-noise negative observation can be determined. When linking with positive observations farther away from this observation, data will be generated in the region where negative observations, which is not a noise, are present. Therefore, the number of positive observations until the first negative observation in nearest neighborhood with "good" data set will give the appropriate number of nearest friends, $k _ { i }$ , for $x _ { i } ^ { p o s }$ .

# Experimental set-up

# 4.1. Data-sets

The two simulation sets which were used in the study differ in how they were simulated. The first set was simulated in such a way as to allow for a linear separation and noise inclusion. Class variable of this two-dimensional data-set had 1000 negative $( N ( 0 , 3 ) )$ and 50 positive $( N ( 6 , 2 ) )$ cases. Then, randomly selected $0 . 3 \times 5 0 = 1 5$ observations were changed to positive in order to generate noise.

Second set was simulated with a similar mindset with the first set but for non-linear classifiers. In order to create a data-set that cannot be separated linearly but non-linearly, $X _ { i }$ ,which consist of 1000 twodimensional samples with $N ( 0 , 1 )$ , was generated. Then $Y$ was generated using equation (6). Obtained variable $Y$ has approximately 1:21 imbalance ratio.

$$

# 4.3. Performance metrics

It is very important to select the performance criterion that will evaluate the model when setting up a classification model in a data-set with class imbalance. The most commonly used criterion for measuring the performance of the classifiers is the accuracy or error rate, which are basically two sides of the same coin. Accuracy measures the percentage of correctly predicted classes, and error rate is the percentage of incorrectly predicted classes. These two criteria can work in many situations. In the case of class imbalance, the accuracy rate is a misleading criterion as it focuses on the negative class rather than the positive class (Bekkar et al., 2013). Researchers may want the positive accuracy to be higher with minimum sacrifice of negative accuracy (Daskalaki et al., 2006). Therefore, it is necessary to look at alternative performance criteria. The most commonly used criteria are balanced accuracy rate, G-mean, $F _ { 1 }$ score, area under ROC (AUC), and Matthews correlation coefficient (MCC) (Matthews, MCC). Among these criteria, only MCC does not change based on the defined class and therefore has advantage over other criteria in this regard for binary classification (Chicco, 2017; Chicco & Jurman, 2020). In this study, MCC criteria is chosen to measure the performance of the models. Equation $( 7 )$ gives the formula to calculate MCC which is calculated using confusion matrix given in Table 3.

$$
M C C = { \frac { T P \times T N - F P \times F N } { \sqrt { ( T P + F P ) ( T P + F N ) ( T N + F P ) ( T N + F N ) } } }
$$

# 6. Conclusion and future works

Many of the existing classification algorithms in machine learning produce a hypothesis, assuming that the number of observations pertaining to classes is balanced. However, this is rarely the case in real data-sets. Imbalance between class observations is called the class imbalance problem. Model performance criteria used in class imbalance problems are different. As already pointed out, MCC was chosen to be used within the criteria along with the reasons for its use. In this study, the problems of ROS and SMOTE methods, which are two of the most frequently used methods in the literature, are discussed and SMOTEWB method is proposed to solve these problems.

In two different simulation data-sets, how ROS and SMOTE methods are affected by noise is presented. This situation and the fact that SMOTEWB is not affected by the presence of noise are shown both in performance and also with visuals. ROS, SMOTE, and SMOTEWB methods were compared in 9 different classifiers in 16 data-sets that are frequently used in the literature. SMOTEWB method has obtained better rank average than other resampling methods in 9 out of 9 classifiers. However only 7 of these differences were found to be significant through Kruskall-Wallis H test. In order to make a general comparison of the SMOTEWB with other resampling methods, a general average MCC ranks of all classifiers was calculated. sMOTEWB has significantly more successful MCC score ranks than other methods. When the obtained results are examined, it can be seen that SMOTEWB method is more successful than ROS and SMOTE methods in data-sets containing noise and class imbalance.

The boosting $^ +$ noise detection method which SMOTEWB uses can be modified for majority class also. This makes allows a new undersampling method similar to SMOTEWB. By combining two of them, it is also possible for a new hybrid method to emerge. We would like to explore more adaptations of SMOTEWB as an undersampling and hybrid method in future.
