# .Introduction

In many real-world classification problems,such as fault diagnosis (Yang, Tang, Shintemirov & Wu, 2009), medical diagnosis (Mazurowski et al., 2008) or credit card fraud detection (Chan & Stolfo, 1998), the number of data in one class is significantly larger than that in other classes, forming class imbalance problems. The class containing a large number of data is called negative or majority class, while the other is positive or minority class (Ali, Shamsuddin, Ralescu, et al., 2015). Without loss of generality, minority data are more important in the classification tasks, and have a higher misclassification cost. For example, faults rarely occur in industrial production process. Thus, the number of normal monitoring data and that of fault ones are skewed. However, we pay more attention on fault ones because they cause the stagnation of production process. Based on this, class imbalance learning has become an open and challenging issue in machine learning that attracts more and more concerns.

When dealing with class imbalance problems, traditional machine learning methods driven by overall accuracy generally find the simplest but sub-optimal classification boundary (Gu, Cai, Zhu, & Huang, 2008). To be specific, the classifiers tend to misclassify the minority data. For example, an imbalance dataset contains 99 majority data and 1 minority one. If we categorize the minority data to negative class, a high classification accuracy about $9 9 \%$ can still be obtained. This is often unexpected due to the importance of detecting the minority events. Therefore, traditional machine learning methods have great limitations in the classification task with imbalanced data. To overcome the above-mentioned drawback, rich studies have been done on class imbalance learning methods that can be categorized into three primary types: data-level, algorithm-level and hybrid approaches (Lim, Goh, & Tan, 2016).

Data-level approaches balance skewed data by oversampling minority data or undersampling majority data (Chawla, Bowyer, Hall, & Kegelmeyer, 2002; Han, Wang, & Mao, 2005; Ng, Hu, Yeung, Yin, & Roli, 2014). We focus on the former in this paper. Random oversampling, as the earliest oversampling method, randomly copies a datum from the minority class (Moreo, Esuli, & Sebastiani, 2016). However, a simple duplication provides uselessly repeatable information for classification, or even results in overfitting. Based on this, the synthetic minority oversampling technique (SMOTE) (Chawla et al., 2002) was presented, in which a new datum is generated by linear interpolation based on minority one randomly selected. After filling it back to the original dataset, a relatively dense and balanced dataset is formed. But SMOTE only concerns the minority data, which may result in a suboptimal decision boundary when the selected minority datum is close to the majority ones. Thus, many improved SMOTE methods were put forward. Both SMOTE-borderline1 and SMOTE-borderline2 pay more attention to minority data close to the boundary and produce a new datum based on the minority ones randomly selected nearby the boundary (Han et al., 2005). Their difference lies in that the latter chooses a part of non-boundary minority data in proportion. SMOTE-SVM took advantage of boundary information derived from support vector machine (SvM), with the purpose of promoting the capability of generalization (Wang, Zou, & Fu, 2021). Different from SMOTEbased oversampling, ADASYN generates new minority samples based on the weighted original minority ones, and the weights are adaptively determined according to the number of majority data around them (He, Bai, Garcia, & Li, 2008). In order to comprehensively utilize information coming from all of classes, radial basis function was employed to learn the data distribution around a minority instance. Based on it, multiclass radial-based oversampling (MC-RBO) (Krawczyk, Koziarski, & Woniak, 2020) was presented to generate new minority samples. Similarly, (Sharma, Bellinger, Krawczyk, Zaiane, & Japkowicz, 2018) sampled the majority for synthesizing information from both minority and majority classes to insert new instances in a meaningful manner.

Being different from the aforementioned methods, algorithm-level approaches solve imbalance classification problems by designing more discriminative strategies, including adjusting the misclassification cost of each category or the classification threshold, as well as the recognition-based methods (Chawla, Japkowicz, & Kotcz, 2004). (Yang, Yang & Wang, 2009) assigned different misclassification costs to positive and negative instances. We call it cost-sensitive imbalance learning. (Kowalczyk & Raskutti, 2002) conducted one-class SVM from positive class instances. (Wang, Huang, Sun, Cao, & Zhao, 2013) introduced cost-sensitive learning to SMOTE, with the purpose of emphasizing the contribution of minority instances on classification. Misclassification costs for instances that have an obvious impact on classification efficiency is generally defined in advance. How to set appropriate misclassification cost is an open challenge (Galar, Fernandez, Barrenechea, Bustince, & Herrera, 2011).

To overcome the shortcomings, hybrid methods provide a framework to combine the above-mentioned two techniques, with the purpose of mining the implicit distribution of data and building a more rational classifier. In particular, considering the superior performance of integrating multiple learning models, ensemble class imbalance learning was proposed (Galar et al., 2011; Malhotra & Lata, 2020). Until now, the most representative ensemble class imbalance learning methods are Bagging and Boosting. The former trains more than one base classifiers in parallel, while the latter generates each classifier in series (Breiman, 1996; Freund, 1995). No matter which kinds of training methods for base classifiers, they all depend on multiple balanced sub-datasets randomly extracted from the original imbalance dataset by resampling techniques. After introducing oversampling or undersampling, UnderBagging, SMOTEBagging, RUSBoost and SMOTE-Boost (Abolkarlou, Niknafs, & Ebrahimpour, 2014; Chawla, Lazarevic, Hall, & Bowyer, 2003; Kang, Chen, Li, & Zhou, 2016; Mounce et al., 2017) were presented. The former two employ the same framework as Bagging, whereas Boosting is introduced to the latter two. Under-Bagging and RUsBoost balance skew data by randomly undersampling, however, SMOTE is employed in SMOTEBagging and SMOTEBoost. To distinguish multiple imbalance classes. (Garcí a, Zhang, Altalhi, Alshomrani, & Herrera, 2018) developed a dynamic scheme called DES-MI to calculate the classifier competences by weighted analysis. Though many efforts have been done on applying various resampling techniques to improve the performance of ensemble class imbalance learning, seldom studies are related to the structure of a multi-classifier model.

To the best of our knowledge, pre-setting the rational scale of a multi-classifier model, generating a base classifier with the better performance and guaranteeing the diversity among them are three key issues in ensemble class imbalance learning. Traditional Bagging provides a powerful framework to sketch the global information to some extent, however, may produce the similar base classifiers by independently training sub-datasets, deteriorating their diversity (Webb & Zheng, 2004). Besides, the number of base classifiers is often determined by human in advance. Too many base classifiers may provide redundant information for classification, increasing the computation cost (Zhu, Wang, Li, Zhu, & Du, 2018). For instance, under the same oversampling strategy, an ensemble model with 50 base classifiers may obtain the similar testing error as the one with 30 base classifiers. That is, more than one combination may have the similar classification performance. Thus, seeking ensemble structures with the highest classification accuracy is essentially a multi-modal optimization problem. To address the issue, multi-modal evolutionary algorithm, as a commonly-used and powerful problem-solver, is introduced to find the combinations with the best classification performance as more as possible. In view of this, a dual evolutionary Bagging (Evo-Bagging) is proposed. Here, the so-called 'dual' refers to two ensemble models nested each other. The inner ensemble model provides base classifiers with optimal parameters for outer ensemble one, and then more than one optimal combinations are found by multi-modal evolutionary algorithm. According to Occam's razor, a classifier with simpler structure usually have better generalization. Thus, the combination that contains the smallest number of base classifiers is chosen as the final ensemble classifier on an imbalanced dataset. Three main contributions are summarized as follows:

1. A dual-ensemble framework composed of inner and outer ensemble models is constructed. The inner ensemble model is designed to seek an optimal base classifier for each sub-dataset obtained by a resampling strategy. Based on the base classifiers available, the optimal combination with the simplest ensemble structure is developed.   
2. A base classifier optimization technique is presented, with the purpose of enlarging the difference among sub-classifiers in inner ensemble model. Three kinds of learning methods, including multi-layer perceptron(MLP), decision tree(DT) and SVM, are employed to build hybrid sub-classifiers and their parameters are optimized by grid search. For each sub-dataset, the subclassifier with the best classification accuracy is selected as a base classifier for outer ensemble model.   
3. In outer ensemble model, multi-modal genetic algorithm based on niche strategy is introduced to seek all optimal combinations of base classifiers in terms of the classification performance. G-mean, as a commonly-used indicator, is employed as the optimization objective. Among the potential optima, the combination aggregating the smallest amount of base classifiers by the weighted sum forms the final ensemble structure.

The rest of this paper is organized as follows. Section 2 formulates the main idea of Evo-Bagging. Section 3 illustrates the base classifier optimization technique in detail. The multi-modal genetic algorithm used to seek the optimal structure for the outer ensemble model is presented in Section 4. Section 5 compares Evo-Bagging with other state-of-the-art methods on 40 benchmark imbalance datasets. A realworld application of Evo-Bagging on classifying rock burst in coal mine is reported in Section 6. Finally, a summary of findings is provided, and several topics to research in the future are given.

# The main idea of Evo-Bagging

In Evo-Bagging, a dual ensemble structure is proposed to construct the optimal ensemble structure for imbalanced datasets. In inner ensemble model, more than one heterogeneous sub-classifier is built for each sub-dataset after oversampling and the optimal one is selected as a base classifier. Following that, the optimal combination of base classifiers is found by multi-modal genetic optimization in outer ensemble model. In view of this, how to design sub-classifiers, as well as build the optimal structure ofa multi-classier are tw key issues of Evo-Bagng. The whole framework of Evo-Bagging is shown in Fig. 1.

The main objective of an inner ensemble model is to generate the most satisfactory base classifier for a sub-dataset after oversampling. To the best of our knowledge, base classifiers in previous ensemble class imbalance learning methods are commonly built by SVM, DT, MLP, ELM, NBM and KNN. Among them, MLP fully represents the high-order information of datasets (Windeatt, Duangsoithong, & Smith, 2011), while DT has the advantages of simple calculation and strong interpretability (Tehrany, Pradhan, & Jebur, 2013). As a strong classifier, SVM provides a powerful supplement for the above-mentioned weak classifiers (Liu, Wu, & Chen, 2010). Based on their performance considerations, MLP, DT and SVM are selected to build sub-classifiers in inner ensemble model, with the purpose of maximizing the difference among sub-classifiers and fully capture the characteristics of imbalanced data. What is more, main parameters of base classifiers are normally set to empirical values or adjusted by human. Though intelligent optimization methods have been introduced to seek the optimal hyperparameters, which provides a full exploration for parameter space, but is high timeconsumption (Lim et al., 2016; Sun et al., 2015). Based on this, grid search is introduced to optimize the parameters of each sub-classifier. Finally, a sub-classifier that has the least testing error is selected as the optimal base classifier for a sub-dataset.

In Evo-Bagging, the number of alternative base classifiers is preset before training. To the best of our knowledge, an ensemble model integrating base classifiers with distinct structures can obtain better classification performance, since different types of learning models are usually associated with various search/representation biases (and thus hypothesis spaces), thereby fomenting ensemble diversity (Coelho & Nascimento, 2010; Nascimento, Canuto, Silva, & Coelho, 2011). More especially, different measurements are designed for the selection of heterogeneous classifiers to ensure the diversity of sub-classifiers in the ensemble model (Kadkhodaei, Moghadam, & Dehghan, 2020; Xia, Liu, Da, & Xie, 2018). Thus, the larger number of alternative base classifiers may produce a good combination with a high probability, contributing to potential higher classification accuracy. Though a large amount of

All neurons between two adjacent layers are fully connected, with the purpose of transferring the features into different scales and combining them into high-order ones. # 3.2. DT-based sub-classifier

DT is presented to mine the rules from an unordered and irregular dataset (Tehrany et al., 2013). Each decision tree consists of a number of leaves and branches. Each non-leaf node represents a feature, and different ranges of the feature values form various branches. In the end, a class corresponds to a leaf node. Based on a decision tree, we start decision-making from the root node, followed by non-leaf nodes, and select the output branch of each node according to the value of the corresponding feature, until the leaf node is reached. The achieved leaf node determines the class of a datum (Djukova & Peskov, 2007; Nugraha, Maulana, & Sasongko, 2020). As a widely-used method to realize the above construction and decision process, CART (Li et al., 2017) is employed in this paper. The importance of each feature is assessed by the Gini index. Suppose that there are $K$ classes. The probability that a datum belongs to the kth class is denoted as $p _ { k }$ , and the corresponding Gini index is defined as follows.

$$
G i n i = 1 - \sum _ { k = 1 } ^ { K } p _ { k } ^ { 2 }
$$

# 3.3. SVM-based sub-classifier

SVM is a typical classification algorithm based on statistical learning theory and structural risk minimization principle (Guo, Zhang, & Tang, 2021). Its goal is to maximize the margin between the optimal hyperplane and the support vector. Let $w$ be a normal vector to the hyperplane, and $b$ be the distance between the hyperplane to the origin. The hyperplane can be described as follows.

$$
\boldsymbol { w } ^ { T } \boldsymbol { x } + \boldsymbol { b } = 0
$$

Assuming that the hyperplane can correctly classify the training datum $( x _ { i } , y _ { i } )$ , the following formula is held:

$$
y _ { i } ( w ^ { T } x _ { i } + b ) \geq 1
$$

Among them, points satisfying $y _ { i } ( w ^ { T } x _ { i } + b ) = 1$ are called the support vector. Margin can be expressed as $d \ : = \ : 2 / | | \omega | |$ . In order to find the maximal margin, the following dual function based on the Lagrange multiplier, denoted as $\alpha _ { i }$ , shall be maximized.

$$
f ( x ) = \sum _ { i = 1 } ^ { M } y _ { i } \alpha _ { i } ( x , x _ { i } ) + b
$$

# 4. The optimal structure for outer ensemble model based on multimodel genetic algorithm

For an ensemble learning model, a reasonable combination of base classifiers is in favor of improving classification accuracy and simplifying its structure. Seeking an ensemble structure with the highest accuracy is essentially a multi-modal optimization problem, because more than one combinations may have the similar classification performance. The key issue of this problem is to seek optimal solutions as more as possible, with the purpose of fully exploring the whole landscape and providing more alternative selection for decision makers. To solve the problem, multi-modal genetic algorithm, as a commonlyused method, finds more than one global or local optimum, and forms a corresponding niche so as to avoid falling into any local one. Thus, multi-model genetic algorithm is introduced to seek the ensemble structures with the highest accuracy as more as possible. These optimal ensemble models may contain different number of base classifiers. Only the one that consists of the smallest number of base classifiers is regarded as the optimal ensemble structure.

Evolutionary algorithms (EAs), as a class of meta-heuristic optimization methods, have a significant advantage in solving non-convex, discontinuous optimization problems (Abu Arqub & Abo-Hammour, 2014; Guo, Zhang, Gong, Zhang, & Yang, 2020; Ji, Guo, Gao, Gong, & Wang, 2021). Rich studies have been done on applying EAs to build the optimal imbalance classifier. On the data level, they have been employed to extract the most useful data, which have an obvious impact on the accuracy. Each datum corresponds to a gene and whether it is assigned for training is labeled with either 0 or 1 in a bit of an individual. Hence, the length of an individual depends on the dataset size. For a large dataset, a binary encoding individual is sparse, which slows down convergence. Being different from it, EAs have been applied to optimize the value of key parameters in a classifier (Liu, Wu, & Zhou, 2008; Wang & Yao, 2013b). (Cheng, 2019) proposed an adaptive CCR-ELM for class imbalance learning, where a variablelength brain storm optimization algorithm is employed to seek the most rational parameters of CCR-ELM. (Luo, Jiao, & Lozano, 2015) constructed an optimal sparse spectral clustering based on a multi-objective evolutionary algorithm. (Pérez-Godoy, Fernández, Rivera, & del Jesus, 2010) optimized radial-based function network by the evolutionary collaboration and competition method. Despite showing more powerful capacity on a single classifier after optimization, it is not enough for superior accuracy and generalization. Based on this, ensemble learning is introduced as a problem-solver for imbalance classification. In order to reduce the computational cost, (Lim et al., 2016) introduced EA to optimize the parameters of a cluster-based synthetic data generation method. (Yang et al., 2013) proposed sample subset optimization techniques for ensemble class imbalanced learning, with the purpose of selecting the most useful samples to sub-dataset. Apparently, previous studies focus on seeking the most reasonable oversampling dataset by optimization, not the best structure of multi-classifier in the ensemble model. Therefore, we develop an outer ensemble model based on a multi-model genetic algorithm to seek the optimal ensemble framework. Its pseudo-code is provided in Algorithm 2.

# 4.1. Objective function

An important issue of multi-model genetic algorithm is the choice of an appropriate objective function for effectively evaluating each individual. In the proposed framework, each individual is translated into an ensemble classifier. Its classification performance on original training dataset is evaluated by G-mean. That is, G-mean (Bhowan, Johnston, Zhang, & Yao, 2014) of each ensemble model is employed as the objective function. Let $x c _ { i } , i = 1 , 2 , \ldots , P _ { s }$ be the ith individual, each gene of $x c _ { i }$ corresponds to a base classifier, represented by $x c _ { i j } , j =$ $1 , 2 , \ldots , L e n$ A set of base classifiers generated from the inner ensemble model is expressed by $C = \{ C _ { 1 } , C _ { 2 } , \ldots , C _ { n } \}$ ,where $n = L e n$ For each base classifier, the corresponding allele in an individual is assigned to 1 if it is employed to build an ensemble model; otherwise, this allele is set to 0. For example, there are five base classifiers built in inner ensemble model, denoted as $C = \{ C _ { 1 } , C _ { 2 } , C _ { 3 } , C _ { 4 } , C _ { 5 } \}$ . An individual, represented by $x c _ { i } = \{ 0 , 1 , 0 , 1 , 0 \}$ , means that the final ensemble model is composed of $C _ { 2 }$ and $C _ { 4 }$ .

# 4.2. Genetic operators

During the evolution, offsprings are produced based on parents by mutation and crossover operators, respectively. In this paper, a single point mutation operation is adopted, with the purpose of avoiding premature convergence. For a randomly selected gene, the opposite binary value is generated to replace the current one in terms of the mutation probability, $p _ { m }$ . Subsequently, a new individual is formed. By the aforementioned operation, a base classifier will join in the ensemble model as the corresponding gene changes from 0 to 1, and vice versa. Similarly, the crossover operator swaps the gene of two parents in terms of the crossover probability, $p _ { c }$ , and forms two offsprings. That means two ensemble models exchange a part of active base classifiers so as to produce two distinct structures.

# 4.3. Niche technique

Without loss of generality, a multi-modal problem may have more than one global (or local) optima. To avoid sinking into premature, a niche technique is developed to maintain the diversity of a population (Li, Balazs, Parks, & Clarkson, 2003). During the evolution, each optimum refers to a niche, and all peaks are stably covered by subpopulations, with the purpose of providing the selection pressure within the local search regions (Wang, Ma, Ju, & Liu, 2010). Rich studies have done on this technique and various niching methods, such as clearing, clustering, crowding and sharing, have been presented (Singh & Deb Dr, 2006). In this paper, the sharing strategy is introduced to divide a population into different sub-populations in terms of the similarity of two individuals. We employ the Euclidean distance to calculate the difference between two individuals. For $x c _ { i }$ and $x c _ { j }$ , their similarity is evaluated by the following distance.

$$
D _ { i j } = \left. x c _ { i } - x c _ { j } \right. _ { 2 }
$$

The niche radius is set to $L$ in advance. When $\begin{array} { r c l } { D _ { i j } } & { < } & { L } \end{array}$ ,the individual with the worst fitness value is penalized by reducing the fitness value. After sorting all individuals in the descending order, the first $P s$ individuals are retained to the next generation, where $P s$ is the population size.

# 4.4. Decision-making of an optimal structure

Due to multi-modal characteristic of outer ensemble model, more than one combination may be obtained from the above-mentioned multi-modal genetic algorithm. Among them, the one containing the larger number of base classifiers may consume higher computation cost. Based on this, we prefer the combination with the simplest structure. Suppose that all optimal solutions form a st $P _ { b e s t }$ , and the size of $P _ { b e s t }$ is $| P _ { b e s t } |$ . The individual $x c _ { i } \in P _ { b e s t }$ containing the smallest number of '1' is defined as the final output $x c _ { o u t }$ of outer ensemble.

That is, the final ensemble model $E$ aggregates the smallest number of base classifiers. Let $\omega _ { i }$ and $\hat { y } _ { i }$ denote the weight and output of the ith base classifier in ensemble $E$ , respectively, and $| E |$ is the ensemble size. The prediction result of Evo-Bagging is the weighted sum of $E$ .

$$
\hat { y } = \sum _ { i = 1 } ^ { | E | } \omega _ { i } \hat { y } _ { i }
$$

where,

$$
\omega _ { i } = { \frac { 1 } { | E | } }
$$

# 5. Complexity analysis

EVO-bagging mainly spends the computational cost on three key steps, including oversampling sub-datasets, generating candidate base classifiers and selecting the optimal combination. Take ROS as an example, the computational complexity of oversampling sub-datasets is $\mathcal { O } ( n d )$ , where $n$ is the number of sub-datasets and $d$ is sub-dataset size. For each balanced sub-dataset after oversampling, three base classifiers are generated and their parameters are optimized by grid search. Let $T _ { p }$ be time-consumption on each $\mathbf { k }$ -fold cross-validation on a sub-dataset, and $P _ { n }$ be the number of candidate parameter combinations, the runtime of generating candidate base classifiers is $\mathcal { O } ( n T _ { p } P _ { n } )$ . The optimal combination of base classifiers is found by multimodal evolutionary optimization. Denoted $P _ { s }$ as population size and $T _ { g }$ as the evaluation cost for a combination, the computational cost for optimization is $\mathcal { O } ( T ( P _ { s } + P _ { s } ^ { 2 } + P _ { s } T _ { g } ) ) _ { : }$ ,in which $\mathcal { O } ( P _ { s } T _ { g } )$ and $\mathcal { O } ( P _ { s } + P _ { s } ^ { 2 } )$ represent the runtime of fitness evaluation and evolutionary operations, respectively. $T$ is the termination iteration. To sum up, the overall computation complexity of EVO-bagging is $\mathcal { O } ( T ( P _ { s } + P _ { s } ^ { 2 } + P _ { s } T _ { g } ) + n d + n T _ { p } P _ { n } )$ .

# 6.1. The indicators of evaluating a method

Traditional metrics that evaluate the performance of a method classifying balanced datasets refer to the overall accuracy of the whole samples, which may ignore the misclassification of minority data for class imbalance learning. Hence, we adopt the following five indicators to provide more insight into not only the overall accuracy of the whole datasets, but also the classification accuracy obtained within each class.

Without loss of generality, the minority and majority classes for binary imbalance classification problems are usually labeled as the positive and negative ones. TP, TN, FP and $F N$ are denoted as the number of data with true positive, true negative, false positive and false negative, respectively. False positive means that a majority sample is misclassified into the minority class, and false negative has the opposite meaning. Accuracy as the most commonly-used metric is defined as follows.

$$
{ \mathrm { z c u r a c y } } = { \frac { T P + T N } { T P + T N + F P + F N } }
$$

Accuracy cannot fully evaluate the classification accuracy of class imbalance learning, especially, for minority data that are misclassified to majority class. To overcome the drawback, G-mean and the area under the curve (AUC) are employed, with the purpose of comprehensively evaluating the accuracy of both the minority and majority classes. The value of AUC is calculated by the area under the receiver operating characteristic (ROC) graph. The ROC graph takes the false positive rate as $X$ -axis and the true positive rate as $Y$ axis. Gmean evaluates the overall classification performance by calculating the geometric mean of recall as follows. The so-called recall refers to the number of minority data being correctly classified, expressed by $T P / ( T P + F N )$ . A larger value of recall may decrease the precision of the classifier and misclassify more majority data.

F-value calculates the harmonic mean of the precision and recall, which is employed to evaluate the classification performance of a method on each class. Precision, denoted as $T P / ( T P + F P )$ , refers to the number of data being correctly classified.

$$

where $\beta$ is a parameter to tradeoff precision and recall, with its value normally being 1 in the paper. Obviously, F-value maximizes the performance on the minority samples by combining precision with recall.

Q-statistic is defined as an indicators measuring the difference between two base classifiers (Wang & Yao, 2009). Let $M$ be the number of the training data. Assuming that $M ^ { 1 1 }$ and $M ^ { 0 0 }$ respectively represent the number of data being correctly classified and misclassified by both classifiers, denoted as $L _ { i }$ and $L _ { k }$ . $M ^ { 0 1 }$ and $M ^ { 1 0 }$ are their number of data being misclassified.

$$
Q _ { i k } = \frac { M ^ { 1 1 } M ^ { 0 0 } - M ^ { 0 1 } M ^ { 1 0 } } { M ^ { 1 1 } M ^ { 0 0 } + M ^ { 0 1 } M ^ { 1 0 } }
$$

For a class ensemble learning model, the average $\mathbf { Q }$ -statistic comprehensively reflects the diversity of all base classifiers. A larger value of $Q _ { a v }$ means a worse diversity among base classifiers (Kuncheva & Whitaker, 2003).

$$
Q _ { \alpha v } = \frac { 2 } { N ( N - 1 ) } \sum _ { i = 1 } ^ { N - 1 } \sum _ { k = i + 1 } ^ { N } Q _ { i , k }
$$

The value of $Q _ { a v }$ varies between $^ { - 1 }$ and 1. Once all base classifiers obtain the correct class for a dataset, $Q _ { a v } > 0 _ { \cdot }$ otherwise, $Q _ { a v } < 0$ .

# 6.2. Comparing the classification accuracy on different types of base classifiers

To verify the rationality of the base classifier optimization technique, we compare it with inner ensemble models composed of homogeneous sub-classifiers built by DT, SVM or MLP. No matter which learning method is employed, three sub-classifiers are built for each sub-dataset.

In inner ensemble model, we build a combination of sub-classifiers for each sub-dataset. However, how many sub-datasets should be generated is an open and challenging issue. Based on this, we further analyze the impact of the sampling rate and the number of sub-dataset on classification accuracy. Fig. 3 depicts the classification performance of inner ensemble model containing different numbers of heterogeneous base classifiers. SMOTE is employed for oversampling and the sampling rate of a sub-dataset is set to $2 0 \%$ , $4 0 \%$ , $6 0 \%$ , $8 0 \%$ and $1 0 0 \%$ respectively. Under each sampling rate, the number of sub-datasets varies from 1 to 60 every 5. The same amount of base classifiers, correspondingly, are generated. Experimental results show that with the increase of the sampling rate, no matter how many sub-datasets are produced, inner ensemble can classify imbalanced data with a higher accuracy. More especially, the classification accuracy reach $9 4 \%$ and holds steady under $1 0 0 \%$ sampling rate. A larger sampling rate means that more minority data are sampled from the original skew dataset, which provides more useful information for sub-classifiers. In addition, under the same sampling rate, inner ensemble has a stable accuracy as the number of sub-datasets exceeds 30. Too small number of subdatasets provide less information about the distribution of imbalanced datasets for sub-classifiers, causing worse diversity of classifiers, even misclassification. Therefore, we set the sampling rate to $1 0 0 \%$ and the number of sub-datasets to 30 in the following experiments.

Without loss of generality, a large difference among sub-classifiers is in favor of improving the accuracy. To compare the diversity of homogeneous and heterogeneous sub-classifiers, five kinds of oversampling methods, including random oversampling (ROS), SMOTE, SMOTE-borderline1, SMOTE-borderline2 and SMOTE-SVM, are applied to generate the balanced sub-dataset. Among them, the latter four are adaptive synthesis minority oversampling techniques. Q-statistics of different inner ensemble models are listed in Table 2, in which 'SMD' represents heterogeneous inner ensemble classifier generated by the proposed Evo-Bagging. Apparently, heterogeneous ensemble classifier has good robustness to all sub-datasets due to its smallest value of Q-statistics.

Under the aforementioned five oversampling techniques, we further compare the structures of homogeneous and heterogeneous subclassifiers on all 40 benchmark datasets. Taking a decision tree-based classifier as the baseline, Fig. 4 depicts average performance improvement of four inner ensemble model under independent 10 experiments. Apparently, SMD with diverse sub-classifiers significantly improve classification accuracy. Moreover, optimizing the parameters of SMD by grid search provides high-qualified local learning models, which contributes to better global classification performance.

# 7. Experiments on a real-world imbalance dataset

Rock burst is a typical disaster in coal mine, which may cause the collapse of a roadway and directly threaten workers' safety. SOS, as a micro-seismic monitoring system, provides the early warning of rock burst (Qi, Liu, Wang, Wang, & Liu, 2011). In a real-world dataset collected from SOS system installed in Yankuang Group, there are 20 minority data and 200 majority ones, and the imbalance ratio is 10. Eight commonly-used classification methods, including SVM, K-Nearest

Neighbor (KNN), MLP, DT, Logistic Regression (LR), Fisher Discriminant Analysis (FDA), Gaussian Bayesian (GB), Stochastic Gradient Descent (SGD) and Bagging, are employed for comparison.

Experimental results shown in Table 12 and Fig. 10 indicate that ensemble classifiers obtained by Bagging and EvO-bagging achieve better classification performance than other comparative methods. In Bagging, each base classifier generates a classification boundary for a sub-dataset after oversampling and all of local information about the dataset are aggregated to obtain a more comprehensive description for the distribution of skewed data. Apparently, the diversity of base classifiers plays an important role in maintaining the satisfactory value of AUC. Based on this, six aforementioned ensemble methods are further compared with Evo-Bagging. We observe from Fig. 11 that Evo-Bagging gains a significant advantage on F1-score, Accuracy, AUC, Recall and G-mean, except for Precision due to the conflict between recall and precision. For a micro-seismic dataset, misclassifying a minority data into majority class means that a large rock burst is mistaken to be safe, resulting in the delay of alarming, even collapse accident of a roadway. Thus, a higher Recall is more important for the classification of a microseismic dataset and Evo-Bagging has a more powerful capacity on it than others.

# 8. Conclusion

Ensemble class imbalance learning that integrates resampling technique with multi-classifier model, provides an effective problem-solver for a skewed dataset. Though it brings better generalization on minority class, the number of base classifiers is preset by human, resulting in redundant structure or poor accuracy. To address the issues, a dual evolutionary bagging that consists of inner and outer ensemble models, termed as EVO-bagging, is presented. In each inner ensemble model, three heterogeneous sub-classifiers are employed to learn the characteristic of a sub-dataset. The one with the highest classification accuracy is selected as a base classifier for it and provided to outer ensemble model. Following that, the combinations with the highest G-mean are found by multi-modal genetic algorithm. Among them, the one that contains the smallest number of base classifiers is regarded as the optimal ensemble structure. Experimental results on 40 KEEL benchmark and a real-world rock burst dataset show that the proposed EvO-bagging outperforms the other seven state-of-the-art ensemble classification methods in all metrics. However, EVO-bagging still suffers drawbacks in imbalanced datasets with extreme IR. How to maintain the diversity between base classifiers in inner ensemble model is an open challenge. Expanding EVO-bagging to classify the extreme imbalance datasets and analyzing the error boundary based on the theories proposed by (Abu Arqub, 2019; Abu Arqub & Rashaideh, 2018) will be our future work. Also, to reasonably set the values of parameters in evolutionary operators, combining adaptive genetic operators (Moyano, Gibaja, Cios, & Ventura, 2020; Moyano & Ventura, 2022) with the proposed dual framework will be our future work.
